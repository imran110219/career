# Contributing to Career Materials

Keep changes factual, reviewable, and small. Do not invent employers, dates, metrics, tools, project details, certifications, or people-management scope. Preserve historical information by moving it to an inventory or archive rather than deleting it. Use `NEEDS REVIEW` whenever evidence is unclear, and update the changelog for meaningful changes.

Before committing, inspect `git diff`, run the appropriate resume build, and avoid adding private files or confidential evidence.
