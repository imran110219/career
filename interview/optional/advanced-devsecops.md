# Advanced DevSecOps

These are secondary DevSecOps topics. They should not delay applications for Senior Backend, Lead Backend or Software Architect roles unless repository evidence or a target job description justifies higher priority.

## Secondary Topics

| Topic | Treatment |
| --- | --- |
| Vault | Learn after secret-management fundamentals. |
| OPA and policy as code | Optional for platform/security-heavy roles. |
| Checkov | Optional Terraform security scanning. |
| Drift detection | Optional IaC operations topic. |
| SLSA | Optional supply-chain maturity topic. |
| Syft and Grype | Optional SBOM/vulnerability scanning tools. |
| In-toto | Optional supply-chain attestation topic. |
| Istio | Optional service mesh topic; avoid unless job requires it. |
| Advanced zero-trust implementation | Optional architecture/security specialization. |
| Advanced cloud governance | Optional for platform or enterprise architect roles. |

## Active Preparation Boundary

Core preparation may include secure CI/CD, SAST fundamentals, dependency scanning, Trivy, secret management, Terraform basics, threat modelling, OWASP, SBOM basics, IAM, audit logging, observability and secure Kubernetes configuration. Everything in this file is P2 unless a job description says otherwise.

## Evidence Status

Learning only unless a lab, project or professional example is added to [../EVIDENCE_LOG.md](../EVIDENCE_LOG.md).
