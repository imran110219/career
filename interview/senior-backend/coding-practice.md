# Coding Practice

Do not turn this repository into a large coding archive. Keep reusable patterns, selected problems, complexity notes and lessons learned.

## Pattern Plan

| Pattern | Priority | Practice outcome |
| --- | --- | --- |
| Arrays and strings | P0 | Indexing, frequency counts, prefix/suffix thinking. |
| Hash maps and sets | P0 | Lookup, deduplication, grouping, collision-aware explanation. |
| Stacks and queues | P0 | Parsing, monotonic stack basics, BFS queue usage. |
| Linked lists | P1 | Pointers, reversal, cycle detection. |
| Binary search | P0 | Sorted data, answer-space search, off-by-one discipline. |
| Trees | P1 | DFS, BFS, recursion, iterative traversal. |
| Heaps | P1 | Top-k, scheduling, priority queues. |
| Sliding window | P0 | Fixed and variable window, frequency constraints. |
| Two pointers | P0 | Sorted arrays, partitioning, in-place movement. |
| Graph traversal | P1 | BFS/DFS, visited sets, connected components. |
| Basic dynamic programming | P1 | State definition, transitions, small common patterns. |
| Complexity | P0 | Time/space explanation and trade-offs. |

## Interview Framing

Explain the brute-force approach briefly, improve with a clear data structure or invariant, state complexity, then code carefully. Prefer simple, correct Java over clever solutions.
