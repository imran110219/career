# Senior Backend Technical Questions

## Implementation Depth

- Walk through how you would implement a secure Spring Boot endpoint from controller to database.
- How would you structure validation, error handling and transaction boundaries?
- How would you debug an endpoint that becomes slow after traffic increases?
- How would you detect and fix an N+1 query problem?
- How would you design Redis caching without returning stale sensitive data?

## Production Scenarios

- A deployment succeeds but users report intermittent failures. What do you check first?
- A PostgreSQL query sometimes blocks for minutes. How do you investigate?
- A payment callback is retried and creates duplicate processing risk. How do you design around it?
- A service's JVM memory keeps growing. What data do you collect?
- Authorization tests pass, but production users see incorrect data. What could be wrong?

## Code Review

- What backend issues do you look for in a pull request?
- How do you review transaction and locking behavior?
- How do you give feedback when a solution is over-engineered?
- What checks would you require before approving AI-generated code?

## Evidence Prompt

For each answer, identify whether the example is professional evidence, personal project evidence, lab evidence or learning only.
