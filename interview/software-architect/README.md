# Software Architect Track

This track prepares for architecture roles while keeping claims tied to verified backend and system-design evidence.

## How To Use

1. Review shared foundations: [system design](../shared/system-design.md), [distributed systems](../shared/distributed-systems.md), [API design](../shared/api-design.md), [security](../shared/security.md), [cloud/DevOps](../shared/cloud-devops.md), and [PostgreSQL](../shared/postgresql.md).
2. Study [architecture patterns](architecture-patterns.md).
3. Practise [system design cases](system-design-cases.md).
4. Prepare [non-functional requirements](non-functional-requirements.md).
5. Use [architecture governance](architecture-governance.md) to discuss decision quality and standards.
6. Run [mock interviews](mock-interviews.md).

## Interview Positioning

- Emphasize system boundaries, data ownership, API contracts, security, scalability, operability and migration strategy.
- Do not overclaim formal architecture governance, long-term platform reliability ownership, or cloud ownership until evidence is captured.
- Use project case studies to keep architecture answers grounded.
