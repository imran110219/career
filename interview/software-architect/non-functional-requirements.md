# Non-Functional Requirements

## NFR Checklist

| Requirement | Interview prompts |
| --- | --- |
| Availability | What uptime target is required and for which workflows? |
| Scalability | Which dimensions scale: users, requests, reports, data, integrations? |
| Reliability | What failure modes are expected and how are they detected? |
| Performance | What latency or throughput target matters to users? |
| Security | What assets, roles, permissions and audit trails are required? |
| Maintainability | How do teams change APIs, schemas and modules safely? |
| Recoverability | What backup, restore, RTO and RPO targets are needed? |
| Auditability | What actions must be attributable and tamper-resistant? |
| Compliance | What legal, privacy or procurement constraints apply? |
| Cost | What cost drivers must be controlled? |
| Operability | What logs, metrics, traces, runbooks and alerts are required? |
| SLO/SLA | Which user-facing promises are measurable? |
| Error budget | How much unreliability is acceptable before feature work pauses? |
