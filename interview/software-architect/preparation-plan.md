# Software Architect Preparation Plan

| Area | Priority | Status | Practice |
| --- | --- | --- | --- |
| Architecture styles | P0 | Reviewing | Compare layered, modular monolith, hexagonal, clean architecture, microservices and event-driven designs. |
| System boundaries | P0 | Practising | Define domains, service boundaries, data ownership and integration points. |
| Non-functional requirements | P0 | Reviewing | Define availability, reliability, performance, security, RTO, RPO, SLO, SLA and cost. |
| Architecture documentation | P1 | Not started | Create C4, sequence, deployment and data-flow outlines. |
| ADRs | P1 | Not started | Write two decision records for case-study trade-offs. |
| Cloud architecture | P1 | Needs evidence review | Design AWS deployment options without claiming professional ownership. |
| Security architecture | P1 | Practising | Threat model one sensitive workflow. |
| Governance | P1 | Needs evidence review | Prepare architecture review and standard-setting examples. |
| Migration strategy | P1 | Reviewing | Plan evolution from coupled services toward clearer boundaries. |

## Evidence Rule

Architecture learning can support interviews, but résumé claims need project-specific evidence in [../EVIDENCE_LOG.md](../EVIDENCE_LOG.md) and the master inventories.
