# Behavioral Preparation

Use factual STAR stories. Do not invent outcomes, metrics, authority, or team scope.

## STAR Structure

| Section | Purpose |
| --- | --- |
| Situation | Context, stakes, and constraints. |
| Task | Your specific responsibility. |
| Action | What you personally did, including technical decisions and collaboration. |
| Result | Verified outcome, reported outcome, or `Evidence needed`. |
| Lesson | What changed in judgment, process, or technical approach. |

## Story Themes

| Theme | Candidate source | Evidence status |
| --- | --- | --- |
| Technical leadership | Election Management System backend delivery | Professional evidence; exact examples need interview preparation. |
| Performance improvement | Exam platform Redis/backend improvements | Résumé-safe 30% response-time reduction; baseline method still useful to capture. |
| Ambiguous requirements | Nomination, reporting, exam workflow, procurement workflow | Professional evidence; details need story framing. |
| Access control | Keycloak-backed authorization and workflow validation | Professional evidence; quantify only after verification. |
| Distributed integration | SMS, payment, identity, master-data, reporting integrations | Professional evidence; incident details need review. |
| Mentoring and code review | Backend team guidance and reviews | Professional evidence; measurable outcome needs capture. |
| Failure or recovery | Deployment troubleshooting or production incident | Needs evidence review. |

## Practice Prompts

- Describe a difficult delivery you led.
- Describe a technical disagreement you resolved.
- How did you balance delivery speed and technical quality?
- How did you communicate technical risk to non-backend stakeholders?
- Describe a code review that changed the implementation direction.
- Describe an incident follow-up and what changed afterward.

## Evidence Notes

Move role-specific stories into [leadership stories](../lead-backend/leadership-stories.md) or project case studies. Keep sensitive client names, URLs, credentials, logs, and unpublished diagrams out of committed files.
