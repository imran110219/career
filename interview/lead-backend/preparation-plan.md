# Lead Backend Preparation Plan

| Area | Priority | Status | Practice |
| --- | --- | --- | --- |
| Requirement clarification | P0 | Practising | Turn ambiguous requirements into scope, APIs, acceptance criteria and risks. |
| Work breakdown and estimation | P0 | Practising | Break one project scenario into backend tasks and dependencies. |
| Code review quality | P0 | Practising | Prepare standards for correctness, security, performance and maintainability. |
| Mentoring | P0 | Needs evidence review | Prepare examples that distinguish support from formal management. |
| Cross-team coordination | P0 | Practising | Use integration examples across frontend, payment, identity, SMS, reporting or master data. |
| Incident leadership | P1 | Needs evidence review | Prepare a verified diagnosis, communication and follow-up story. |
| Technical-debt management | P1 | Reviewing | Explain when to fix now, contain, or document for later. |
| Release planning | P1 | Reviewing | Cover rollout, rollback, validation, monitoring and stakeholder updates. |

## Weekly Routine

- Practise one STAR story and one delivery scenario.
- Review one shared technical topic so leadership answers remain technically grounded.
- Update [../EVIDENCE_LOG.md](../EVIDENCE_LOG.md) when a story becomes verified and résumé-safe.
