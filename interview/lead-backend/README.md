# Lead Backend Engineer Track

This is the highest-priority track. It emphasizes hands-on backend leadership: requirement clarification, planning, estimation, risk identification, code review, mentoring, coordination, incident leadership, release planning and delivery ownership.

## How To Use

1. Keep technical depth fresh through [shared](../shared/).
2. Follow [preparation plan](preparation-plan.md).
3. Build factual STAR stories in [leadership stories](leadership-stories.md).
4. Practise [delivery scenarios](delivery-scenarios.md).
5. Run [mock interviews](mock-interviews.md).

## Interview Positioning

- Present leadership as technical delivery ownership, not formal people management unless verified.
- Separate personal actions from team outcomes.
- Use Penta Global election-platform delivery as the primary current-role source, with disclosure limits respected.
