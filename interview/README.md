# Interview Preparation

This folder is a Markdown-first preparation system for three target roles, in priority order:

1. Lead Backend Engineer
2. Senior Backend Engineer
3. Software Architect

It separates shared technical preparation from role-specific interview positioning, project case studies, optional differentiation topics, and evidence tracking.

## Start Here

1. Review [PROGRESS.md](PROGRESS.md).
2. Pick one P0 item from [ROADMAP.md](ROADMAP.md).
3. Study the linked shared material under [shared](shared/).
4. Practise relevant prompts in [QUESTION_BANK.md](QUESTION_BANK.md).
5. Implement or document the practical deliverable.
6. Record evidence in [EVIDENCE_LOG.md](EVIDENCE_LOG.md).
7. Update the matching project case study in [projects](projects/).
8. Promote skills to `master/skills-inventory.md` or `resumes/` only after evidence is verified.

## Core Files

| File | Purpose |
| --- | --- |
| [ROADMAP.md](ROADMAP.md) | Milestone-based preparation plan: learn, practise, implement, document, and demonstrate. |
| [PROGRESS.md](PROGRESS.md) | Current dashboard, 30-day focus, weekly priorities, blockers, and next evidence to capture. |
| [EVIDENCE_LOG.md](EVIDENCE_LOG.md) | Practice, lab, personal project, professional, verified, and résumé-safe evidence records. |
| [QUESTION_BANK.md](QUESTION_BANK.md) | Curated high-value questions for technical, leadership, architecture, and candidate-side interviews. |

## Shared Preparation

[shared](shared/) contains common preparation for [Java](shared/java.md), [Spring Boot](shared/spring-boot.md), [PostgreSQL](shared/postgresql.md), [testing](shared/testing.md), [API design](shared/api-design.md), [distributed systems](shared/distributed-systems.md), [system design](shared/system-design.md), [security](shared/security.md), and [cloud/DevOps](shared/cloud-devops.md). Shared files hold reusable technical depth so role folders stay focused on interview expectations and presentation.

## Role Tracks

| Track | Purpose |
| --- | --- |
| [lead-backend](lead-backend/README.md) | Delivery leadership, mentoring, planning, incident leadership, code review, and backend ownership. |
| [senior-backend](senior-backend/README.md) | Hands-on Java/Spring/PostgreSQL depth, coding practice, debugging, performance, security, and medium-level system design. |

## Project Case Studies

[projects](projects/) contains interview case studies for verified or evidence-backed projects. These files are for preparation and storytelling; public implementation proof belongs in [../portfolio](../portfolio/), verified skills belong in [../master/skills-inventory.md](../master/skills-inventory.md), and selected résumé claims belong in [../resumes](../resumes/).

## Optional Topics

[optional](optional/) contains supporting differentiation topics: AI-assisted development, AI security, LLMOps, and advanced DevSecOps. These topics should not delay applications for the three target roles unless a job description makes them central.

## Evidence Flow

Learning notes and labs start in this folder. Public, inspectable project evidence belongs in [../portfolio](../portfolio/). Verified career facts belong in [../master](../master/). Résumés under [../resumes](../resumes/) should use only role-relevant, defensible claims.
