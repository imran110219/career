# Projects

Use this file for professional, open-source, portfolio, and internal projects that may be referenced in resumes or interviews.

## Project Template

| Field | Notes |
| --- | --- |
| Name | Project or system name. |
| Problem | What user, business, or engineering problem it solved. |
| Role | Your ownership and contribution. |
| Stack | Languages, frameworks, database, infrastructure, and tools. |
| Architecture | Key design decisions and tradeoffs. |
| Impact | Metrics, adoption, reliability, speed, or cost. |
| Link | GitHub, live site, article, demo, or internal reference. |

## Project Selection Criteria

- [ ] Demonstrates depth in the target role.
- [ ] Has a clear problem and outcome.
- [ ] Can be discussed technically at multiple levels.
- [ ] Has clean documentation or supporting artifacts.
- [ ] Avoids exposing confidential employer or client information.
